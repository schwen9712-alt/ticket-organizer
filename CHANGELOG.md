## 2026-05-23 — 修复「乘机人：序号+姓名+护照号码」格式解析错误

**改了什么**
修复 `乘机人： 1.DENG/YIRUI  护照号码CN/EG7029264/CN/15OCT00/F/01JUL29` 这种格式的乘客解析错误。

**为什么改**
用户反馈：上一版补了城市映射后航段能识别了，但乘客信息解析错了（看截图）：
- LAST NAME: **1.DENG** ← 序号"1."粘进姓里
- FIRST NAME: **YIRUI 护照号码CN** ← 名字后粘了"护照号码CN"
- DOB / GENDER: **(待补全)** ← 证件信息（15OCT00/F/01JUL29）完全没解析

根因：`乘机人：` 行的解析分支（cnPaxMatch）只认两种子格式：
1. `P/CN/...` SSR 开头
2. `姓名/名 + 日期 + 性别` 结尾

用户的格式既有**序号 1.** 又有**护照号码前缀**，两种都不匹配 → 落到 fallback 把整串当姓名存了 → 姓名带序号和护照号码，证件全丢。

**修复方案**
在 cnPaxMatch 最前面加两个新分支：
1. **序号+护照格式**：`1.DENG/YIRUI 护照号码CN/护照号/国籍/DOB/性别/到期`
   - 剥离序号 1.
   - 提取姓名（护照号码前）
   - 解析证件块（国籍/护照/国籍/DOB/性别/到期）
2. **无序号护照格式**：`DENG/YIRUI 护照号码CN/...`（同上但无序号）

姓名支持空格（DELA CRUZ 等），证件提取国籍/护照/出生/性别/护照到期。

**修复效果对比**
| 字段 | 修复前 | 修复后 |
|---|---|---|
| 姓名 | 1.DENG / YIRUI 护照号码CN ❌ | DENG/YIRUI ✓ |
| DOB | 待补全 ❌ | 15OCT00 ✓ |
| 性别 | 待补全 ❌ | 女 ✓ |
| 护照到期 | 无 | 01JUL29 ✓ |

**端到端测试通过**
- ✓ 序号+护照格式：DENG/YIRUI, 生15OCT00, 女, 到期01JUL29
- ✓ 无序号格式：DENG/YIRUI（也支持）

**改动位置**
- index.html 9438 附近：cnPaxMatch 加 numberedDoc + namedDoc 两个分支

**风险或注意事项**
- ⚠ "护照号码"前缀的几种写法都支持（护照号码/护照号/护照）
- ⚠ 证件格式假设是 国籍/护照号/国籍/出生/性别/到期（双国籍字段）。这是这种 OTA 的标准格式
- ⚠ 不影响其他乘客格式（SSR、标准证件行、纯中文名等）

**回滚方式**
从 .backups/ 找上一版覆盖。回滚后这种格式的乘客姓名/证件又会解析错。
