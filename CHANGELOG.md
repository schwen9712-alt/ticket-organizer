## 2026-05-23 — 修复儿童乘客「儿童标记在性别前」漏识别

**改了什么**
乘客解析支持"儿童/婴儿"标记插在姓名和性别之间，且性别可紧贴日期。

**为什么改**
用户反馈这段没识别到位：
```
1.WANG/YAZHEN 女  28AUG1989
2.LIU/XINYUAN   男    13MAR1986
3.LIU/QINGHE   儿童  女10JAN2022     ← 漏识别
4.WANG/JIAYAN  儿童 男 24JUL2015     ← 漏识别
1.  DL038 Z SA18JUL PVGLAX ...
TOTAL CNY 17365 成人运价
TOTAL CNY 16845 儿童运价
```

诊断结果：
- ✓ 成人（行1/2）正常识别
- ✓ 价格（成人运价/儿童运价 TOTAL CNY）正常
- ✗ **两个儿童（行3/4）漏识别**：格式是 `姓名 + 儿童 + 性别 + 日期`，"儿童"插在姓名和性别之间。现有 regex 期望"姓名+性别+tail"，"儿童"卡在中间导致不匹配
- 行3 "女10JAN2022" 性别还紧贴日期（无空格）

**修复方案**
cnInlinePax regex 改进：
1. 在姓名和性别之间，容纳可选的"儿童/婴儿/成人/大人"标记
2. 性别和日期之间的空格改为可选（支持"女10JAN2022"紧贴）
3. 乘客类型从"显式标记(儿童)"或"性别后缀(女童)"任一识别

**端到端测试（用户输入）全部通过**
- ✓ 4 个乘客全识别（之前只识别 2 个成人）
- ✓ WANG/YAZHEN 女 28AUG89（4位年份1989自动转2位）
- ✓ LIU/QINGHE 儿童 女 10JAN22（儿童标记在性别前 + 性别紧贴日期）
- ✓ WANG/JIAYAN 儿童 男 24JUL15
- ✓ 成人运价 ¥17365 + 儿童运价 ¥16845

**防误伤测试通过**
旧格式仍正常：
- ✓ "WU/XINER 女 25 DEC1993"（4位年份+空格）
- ✓ "WANG/XIAO 男 18JAN95"（紧凑2位年份）
- ✓ "LI/MING 女童 10JAN20"（性别后缀童）
- ✓ "RAO/XIANG 男 29NOV77"

**改动位置**
- index.html 9930 附近：cnInlinePax regex 容纳儿童标记在性别前 + 性别紧贴日期

**风险或注意事项**
- ⚠ "儿童/婴儿/成人/大人"标记位置现在更灵活（性别前），不影响原"性别后缀(女童/男婴)"的识别
- ⚠ 成人/儿童运价（TOTAL CNY 分档）已能识别。具体每个乘客用哪档价，建议录入后核对
- ⚠ 不影响其他乘客格式

**回滚方式**
从 .backups/ 找上一版覆盖。回滚后"儿童"标记在性别前的儿童乘客又会漏识别。
