## 2026-05-23 — 修复关联订单顺序标签的歧义（→名字 被误读）

**改了什么**
把订单行的出票顺序徽章从 `🔗 第1出 → HE` 改为 `① 先出`（圆圈数字 + 先出/后出），关联的乘客名移到悬停提示。

**为什么改**
用户反馈：「明明是先出往返的 PENG 和 HU，怎么识别出先出 HE 了？」

排查发现：**编号其实是正确的**——
- PENG/HU（往返）= ticketOrder 1 = 第1出 ✓
- HE（单程）= ticketOrder 2 = 第2出 ✓
跟文本"先开往返 再开下面单程"完全一致。

**真正的问题是 UI 表达有歧义**：旧标签 `🔗 第1出 → HE` 里的箭头 `→ HE` 被误读成"HE 第1出"。但代码本意是"我(PENG)第1出，关联对象是 HE"——箭头指向的是【关联对象】，不是【谁先出】。这个设计完全是误导。

**修复后的表达**
- PENG/HU：`① 先出`（紫底，悬停看"关联订单：HE"）
- HE：`② 后出`（紫底，悬停看"关联订单：PENG"）

用圆圈数字①②③ + 先出/后出 文字，一眼就懂谁先谁后，不再有箭头歧义。关联谁放在 tooltip（点击还能编辑）。

**改动位置**
- index.html 11702 附近：linkedBadge 手动关联部分重写，去掉 "→名字" 箭头，改用 ①②+先出/后出

**风险或注意事项**
- ⚠ 关联的乘客名现在不在订单行直接显示了，需悬停查看。但"谁先出"是更关键的信息，名字是次要的，这个信息层级更合理
- ⚠ 圆圈数字支持到 ⑨（9单关联），超过显示 (N)
- ⚠ 不影响 👤 自动关联徽章（那个本来就是显示同名乘客，没有顺序歧义）

**回滚方式**
从 .backups/ 找上一版覆盖。回滚后又会显示有歧义的 "🔗 第1出 → 名字"。
