## 2026-05-23 — 一口价可在待出票列表直接改（出票中沟通场景）

**改了什么**
待出票列表的订单卡片现在可以直接设置/修改一口价，不用回录入表单。

**为什么改**
用户需求：「也需要加在待出票的地方，因为很多时候是出着出着在沟通」

真实工作流：订单已经在待出票列表，正在和代理/客户沟通价格，谈定一口价。之前一口价开关只在录入表单，得切回去改，打断流程。

**怎么用（待出票列表里）**
**非一口价订单**：售价区底部有「💰 改用一口价」链接
- 点击 → 自动切到一口价模式，种子价 = 当前收入（数字不跳变）→ 立即弹出输入框
- 输入谈好的金额 → 回车确认

**一口价订单**：
- 点金额 `¥XXXX` → 内联编辑修改
- 「总价⇄/每人价⇄」小标签 → 一键切换总价/每人价
- 「↩ 改回折扣计价」→ 切回原折扣模式
- 编辑时清空金额 = 也能取消一口价

**新增函数**
- startEditFlatPrice / commitEditFlatPrice — 内联编辑一口价金额
- switchToFlatPrice — 折扣订单切到一口价（种子=当前收入，防跳变）
- switchToDiscount — 一口价切回折扣
- toggleFlatPerPax — 总价/每人价切换
- _afterFlatPriceChange — 统一重算+重渲染+云同步

**单元测试 8/8 全过（模拟真实工作流）**
- ✓ 折扣订单 8028
- ✓ 改用一口价（种子=8028 不跳变）
- ✓ 改成谈好价 7500
- ✓ 切每人价（7500×2=15000）
- ✓ 改回折扣（恢复 8920×90%×2=16056）

**改动位置**
- index.html 18263 附近：一口价分支加内联编辑+切回折扣+总价每人切换
- index.html 18347 附近：非一口价订单加「💰 改用一口价」入口
- index.html commitEditDiscount 前（约 17390）：加 5 个一口价交互函数

**设计细节**
- switchToFlatPrice 用当前收入做种子价，避免切换瞬间金额变 0 或 null
- 改一口价走 persistOrdersNow → 立即云同步（多设备一致）
- 改完自动刷新利润、USD汇总面板

**风险或注意事项**
- ⚠ 一口价现在有两个入口：录入表单（新订单）+ 待出票列表（出票中）。两处改的是同一字段，一致
- ⚠ 「改用一口价」种子价是当前折扣算出的收入，点了之后记得改成实际谈定的价
- ⚠ 待出票列表里编辑一口价金额时，清空 = 取消一口价回到折扣

**回滚方式**
从 .backups/ 找上一版覆盖。回滚后待出票列表不能改一口价（只能在录入表单改）。
