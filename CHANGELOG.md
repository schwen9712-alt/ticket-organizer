## 2026-05-23 — 识别「数字+中文描述」无关键词价格（如 32041 商务大人 4位）

**改了什么**
新增对"价格在前+中文舱位/人群描述"格式的识别，这种格式没有 元/CNY/含税 等价格关键词。

**为什么改**
用户反馈这段价格没识别：
```
32041    商务大人   4位
32000  儿童价格
```

诊断：现有所有价格 regex 都要求价格带关键词标识（元/CNY/RMB/含税/TOTAL）。但这种 OTA 简写格式是"大数字打头 + 中文描述"，没有任何价格关键词，所以全部漏识别。

难点：32041 和 "4位"的4 都是数字，怎么区分？→ 价格是 4-6 位大数字，"N位/N人"是人数。

**修复方案**
新增分支：行首 4-6 位数字 + 中文舱位/人群描述词（大人/成人/儿童/婴儿/价格/商务/经济/头等/公务/超经）。
- 数字 = 价格
- 描述含"儿童/婴儿" → 记为儿童/婴儿价（分档）
- 否则 → 大人价（主运价）
- 从描述顺带识别舱位（商务大人 → 商务舱）
- 复用已有的 parsedFareByType.adult/child/infant 分档基础设施

**端到端测试（用户输入）通过**
- ✓ 大人价 32041 → 主运价
- ✓ 儿童价 32000 → 单独记录（分档）
- ✓ 舱位 → 商务舱（从"商务大人"推出）
- ✓ 行李行"1PC 返程2pc"正确跳过

**防误伤测试通过**
这些不会被误判为价格：
- ✓ 年份"2024年5月"
- ✓ 时间"1655 1340"
- ✓ 标准 CNY 格式"13585 CNY"（走原有分支）
- ✓ 行李"1PC 返程2pc"

**改动位置**
- index.html 10046 附近：加 bareNumFare 分支（数字+中文描述价格）

**关于分档价格的使用**
大人价作为主运价（rmb），儿童价存入 parsedFareByType.child。如果这个订单有大人也有儿童，系统会记录两档价格。具体每个乘客用哪档，建议录入后核对（尤其儿童乘客如 JUSTIN/ISABELLE）。

**风险或注意事项**
- ⚠ 必须有中文舱位/人群描述词才匹配，纯"数字+数字"不会被当价格（避免误抓时间/数量）
- ⚠ "N位/N人"识别为人数，不当价格
- ⚠ 大数字范围限定 500-500000，过滤明显非价格的数字
- ⚠ 不影响标准带关键词的价格格式（元/CNY/含税等仍走原分支）

**回滚方式**
从 .backups/ 找上一版覆盖。回滚后这种无关键词价格格式又会漏识别。
