## 2026-06-08 — 新增护照信息卡（分行）乘客识别

**改了什么**
识别"护照信息卡"格式：姓名、性别、出生日期分别在独立行的乘客信息。

**为什么改**
用户需求：增加这种格式的识别：
```
GAO,PENG
男/M
21 MAY 1989
```

这是护照信息卡格式，特点：
- 姓名用**逗号**分隔（GAO,PENG），不是斜杠
- 性别单独一行（男/M）
- 出生日期单独一行，带空格（21 MAY 1989）

现有解析都要求姓名用斜杠、且性别/生日在同一行，所以这种分行格式完全识别不到。

**修复方案**
在解析前加"护照卡多行合并"预处理：
1. 检测"SURNAME,GIVEN"逗号姓名行
2. 往下看 1-3 行，提取性别行（男/M、女/F、男、女、M、F）和出生日期行
3. 合并成标准格式 "GAO/PENG 男 21MAY89"，交给现有乘客解析器

**支持的子格式**
- 姓名：GAO,PENG / LI,XIAO MEI / VAN DER BERG,JOHN（姓名可含空格）
- 性别：男/M、女/F、男、女、M、F、MALE、FEMALE、性别：男
- 日期：21 MAY 1989（带空格）、21MAY89、1989-05-21、1989年5月21日

**端到端测试通过**
- ✓ GAO,PENG + 男/M + 21 MAY 1989 → GAO/PENG 男 21MAY89
- ✓ 多个乘客分别合并
- ✓ 只有性别没生日也处理
- ✓ 中文生日（1988年7月12日）也支持
- ✓ 合并后能被乘客解析器正确识别

**防误伤（双重保护）**
1. NON_NAME 词表排除：INCL/TAX/TOTAL/FARE/CNY/ECONOMY/ADULT/CHILD 等含逗号的非姓名行被跳过
2. 必须后面跟性别行或日期行才合并：孤立的逗号行不会被误认

测试：
- ✓ "INCL TAX, FEES" → 跳过
- ✓ "TOTAL, CHILD" → 跳过
- ✓ "GAO,PENG" → 识别为姓名

**改动位置**
- index.html 9444 附近：expandedLines 构建后加护照卡多行合并预处理

**风险或注意事项**
- ⚠ 逗号姓名行必须紧跟性别/日期行才会被合并，避免误伤普通逗号文本
- ⚠ NON_NAME 词表覆盖常见票务术语，若遇到罕见词误判可继续补
- ⚠ 姓名转为标准斜杠格式（GAO,PENG → GAO/PENG）存储

**回滚方式**
从 .backups/ 找上一版覆盖。回滚后这种分行护照卡格式又会识别不到。
