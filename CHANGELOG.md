## 2026-05-23 — 新增「一口价」结算方式（固定金额销售价）

**改了什么**
出票表单新增「💰 一口价」选项：用固定 CNY 金额作为对代理的销售价，覆盖默认的"原价×折扣"公式。

**为什么改**
用户需求：「目前都是9折或各种折扣作为对代理的销售价，有时候也会以一个一口价来作为结算价」

之前销售价只能走折扣公式（basePrice × discount% × pax）。但有时谈的是一口价（不管原价多少，就收这个固定数），没法表达。

**怎么用**
出票表单价格区，折扣下方新增「💰 一口价（固定金额，覆盖折扣）」开关：
- 勾选后：出现「一口价 ¥」输入框 + 「每人价（×人数）」勾选框
- 原价/折扣输入框自动变灰禁用（因为被覆盖了）
- 实时显示公式：
  - 总价模式：「总售价 ¥13,000（2 人共此价）」
  - 每人模式：「每人 ¥6,500 × 2 人 = 总售价 ¥13,000」

**两种一口价**
- **总价**（默认）：填的数就是全部乘客的总售价
- **每人价**（勾选"每人价"）：填的数 × 人数 = 总售价

**计算逻辑（computeFinalPrice 优先级）**
1. 一口价模式开 + 金额>0 → 用一口价（总价 or 每人×人数）← 最高优先级
2. 否则走原有的折扣公式（paxPrices×折扣 或 basePrice×折扣×人数）

一口价覆盖折扣，但保留安全回退：如果开了一口价但没填金额（或填0），自动回退到折扣公式，不会算出 null。

**单元测试 6/6 全过**
- ✓ 一口价总价（2人共13000）
- ✓ 一口价每人（6500×2=13000）
- ✓ 一口价覆盖折扣公式
- ✓ 关闭一口价→走折扣
- ✓ 一口价没填金额→回退折扣
- ✓ 一口价填0→回退折扣

**新增字段**
- flatPriceMode（一口价开关）
- flatPriceCny（一口价金额）
- flatPricePerPax（true=每人价，false=总价）

**改动位置**
- index.html 5592 附近：schema 加 3 个一口价字段
- index.html 3314 附近：出票表单加一口价 UI（开关+金额+每人价）
- index.html computeFinalPrice（约 8344）：优先用一口价
- index.html toggleSpecialDiscount 前（约 8362）：加 toggleFlatPriceUI 函数

- index.html loadCurrToDOM（约 8361）：回填一口价字段

**影响范围**
- 一口价作为销售价（computeFinalPrice）会自动影响：利润计算、USD换算、汇总、导出、复制报表——因为它们都基于 computeFinalPrice
- 成本/结算价（computeSettlement）不受影响，仍按积分/刷卡/直接折扣等原逻辑

**风险或注意事项**
- ⚠ 一口价是 CNY 销售价。USD 约价会用一口价 ÷ 汇率自动算
- ⚠ 一口价模式下原价/折扣输入被禁用变灰，取消勾选即可恢复
- ⚠ 一口价覆盖折扣，但不影响表Y的9折标记本身（只是9折公式被一口价盖过）。如果一个表Y订单设了一口价，按一口价算
- ⚠ 每人价 vs 总价别搞混：2人订单填6500，总价模式=6500总，每人模式=13000总。UI有实时公式提示

**回滚方式**
从 .backups/ 找上一版覆盖。已存的一口价字段会被忽略（回退到折扣）。
