## 2026-05-23 — 记录员报表补全现金补差显示

**改了什么**
记录员报表的「结算」行加上现金补差金额（如果 cashUsd > 0）。

之前（bug 状态）：
```
结算: 504,388 × 0.65 × 1.34CPP [AMEX BP]
```

现在（cashUsd = 179.05）：
```
结算: 504,388 × 0.65 × 1.34CPP + $179.05 现金 [AMEX BP]
```

无现金补时显示**完全不变**，向后兼容。

**为什么改**
上一版做积分+现金混合支付时，只改了「成本计算」和「UI 公式预览」，**漏了记录员报表的文案**。用户截图反馈：表单底栏显示 `504,388 × 0.65 × 1.34CPP + $179.05 = $4572.27`，但点「📋 记录员报表」复制出来的文本里没有 $179.05 这一项 → 记录员/财务对账时会缺数据。

**改动位置**
`index.html` 行 17800 附近：`buildInternalCopy` 的 usesPoints 路径和 legacy fallback 路径都加 cashSuffix

**风险或注意事项**
- ⚠ 只改记录员报表（internal）。代理商报表（client）是 buildClientCopy，原本就不显示成本细节，不需要改
- ⚠ cashUsd === 0 或 null 时 cashSuffix 是空串，输出和老版本完全相同

**回滚方式**
从 `.backups/` 找上一次的 `index.html` 覆盖即可。100% 安全回滚。
