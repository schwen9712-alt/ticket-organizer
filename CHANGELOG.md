## 2026-05-23 — 视觉系统审计 + 统一徽章系统（设计债清理第一步）

**做了什么**
1. **视觉系统审计**（诊断现状）
2. **建立统一设计 token + ui-badge 徽章系统**
3. **订单行 7 个徽章全部迁移到新系统**

**审计发现的混乱程度**
| 维度 | 审计前 | 健康值 |
|---|---|---|
| 硬编码颜色 | 225 种 / 537 处 | ≤20 |
| 圆角 | 32 种 | ≤4 |
| 字号 | 35 种 | ≤8 |
| padding | 129 种 | ≤10 |
| 徽章 | 71 个全手写内联样式 | 1 套组件 |

徽章是重灾区：71 个徽章用了 10 种圆角 + 8 种字号 + 12 种 padding，每个都是手写魔法数字。这就是订单行视觉不齐、加徽章就撑爆的根因。

**建立的设计系统**
新增 CSS token（单一真相源）：
- 语义色：--purple/--gold/--chase-blue/--neutral-chip 等（收编高频硬编码色）
- 徽章几何：--badge-h(18px)/--badge-pad-x/--badge-radius/--badge-font/--badge-gap

新增 `.ui-badge` 类系统（14 条规则）：
- `.ui-badge` 基座：统一高度/圆角/字号/对齐/gap
- `.ui-badge--icon` 纯图标方形变体
- `.ui-badge--clickable` 可点击
- 状态修饰类：--y(表Y金) / --seq(顺序红) / --link(关联紫) / --autolink(自动蓝) / --hold(放位琥珀) / --none(无位灰) / --mix(混舱) / --chase / --repeat / --warn

**订单行徽章迁移（7个）**
全部从手写内联样式 → ui-badge 类：
- 混舱、🔒放位、⏸无位、📋顺序备注、🔗关联(手动)、👤关联(自动)、⚡表Y

效果：所有徽章现在同高(18px)、同圆角、同字号、同对齐、同间距。订单行无论带几个徽章都视觉整齐，不会再出现某几行特别挤的情况。

**测试验证（jsdom 渲染）**
- ✓ ui-badge 类正确应用
- ✓ 所有徽章高度统一 18px（表Y圆形、无位胶囊是有意差异化）
- ✓ 字号统一（图标类 11px 例外）
- ✓ 订单行硬编码紫/蓝色已清零

**为什么是"第一步"而非全部**
全局替换 225 种颜色风险极大（很多在 modal/图表/表格等未充分测试的地方）。采用渐进式策略：
- 先统一最影响日常视觉的"订单行徽章"（已完成，零破坏风险）
- 建立了 ui-badge 系统 + 语义色变量基础设施
- 其他界面以后用到时逐步迁移到这套系统

这样既建立了正确的设计系统，又不冒"大爆改全挂"的风险。

**风险或注意事项**
- ⚠ 只迁移了订单行徽章。其他界面（出票表单、卡主弹窗、账单、图表）的徽章/颜色暂未统一，仍是旧样式。视觉上订单行会比其他地方更规整，这是渐进迁移的正常中间态
- ⚠ ui-badge 的几个差异化（表Y圆形、无位胶囊）是有意保留的视觉区分，不是不一致
- ⚠ 如果以后要继续统一，下一步建议：出票表单 → 卡主弹窗 → 各 modal，每次一个区域，每次测试
